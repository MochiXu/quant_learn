## 一、什么是有效前沿？

你有 9 个资产，可以给每个资产分配不同比例的资金，叫做**权重（weights）**，权重之和必须等于 1。

```
例：SPY 30%，TLT 25%，GLD 15%，IWM 10%，其余各 5%
weights = [0.30, 0.25, 0.15, 0.10, 0.05, 0.05, 0.05, 0.025, 0.025]
```

不同权重组合出来的风险/收益各不相同。把所有可能的最优组合画成图，形成一条曲线，就是**有效前沿**：

```
收益
 ↑
 │              ╭────  ← 有效前沿（曲线上每个点 = 一种最优权重组合）
 │           ╭─╯
 │        ╭─╯    ← 曲线上方不可达
 │     ╭─╯  ···  ← 曲线下方是"低效"组合（同样风险但收益更低）
 │    ╯     ★    ← 夏普比率最高的点（最优组合）
 └──────────────→ 风险（波动率）
```

**有效前沿的意义**：给定你愿意承受的风险水平，曲线告诉你理论上能达到的最高收益；或者给定目标收益，告诉你最低需要承受多少风险。

## 二、计算单个组合的收益和风险

```python
def portfolio_performance(weights, returns):
    portfolio_return = np.sum(returns.mean() * weights) * 252
    portfolio_volatility = np.sqrt(np.dot(weights.T, np.dot(returns.cov() * 252, weights)))
    return portfolio_return, portfolio_volatility
```

### 2.1 收益率：逐元素相乘

```python
np.sum(returns.mean() * weights) * 252
```

`returns.mean()` 返回长度为 9 的向量，`weights` 也是长度为 9 的向量，`*` 是**逐元素相乘**：

```
资产      日均收益    权重      乘积
SPY       0.0004  ×  0.30  =  0.00012
TLT       0.0001  ×  0.25  =  0.000025
GLD       0.0002  ×  0.15  =  0.000030
...（共9项）

np.sum(...) = 全部加起来 = 组合日均收益率
× 252       = 年化
```

### 2.2 方差协方差矩阵 `returns.cov()`

在计算组合波动率之前，需要先理解这个矩阵。

**协方差（Covariance）衡量两个资产涨跌的联动程度**：

$$\text{Cov}(A, B) = \frac{\sum_{i=1}^{n}(r_{A,i} - \bar{r}_A)(r_{B,i} - \bar{r}_B)}{n-1}$$

用 3 天数据举例（简化为 2 个资产）：

```
日期       SPY 收益    TLT 收益
Day 1      +0.03      -0.01
Day 2      -0.02      +0.02
Day 3      +0.01      -0.01

SPY 均值 x̄_A = (0.03 - 0.02 + 0.01) / 3 = 0.0067
TLT 均值 x̄_B = (-0.01 + 0.02 - 0.01) / 3 = 0.0

第一步：逐日计算偏差之积
Day 1: (0.03 - 0.0067) × (-0.01 - 0.0) = 0.0233 × (-0.01) = -0.000233
Day 2: (-0.02 - 0.0067) × (0.02 - 0.0)  = -0.0267 × 0.02  = -0.000534
Day 3: (0.01 - 0.0067) × (-0.01 - 0.0)  = 0.0033 × (-0.01) = -0.000033

第二步：求和除以 n-1
Cov(SPY, TLT) = (-0.000233 - 0.000534 - 0.000033) / 2 = -0.0004

负值 → 两资产反向波动（股票跌时债券涨），符合现实认知。
```

**`returns.cov()` 输出是一个 9×9 的矩阵（协方差矩阵 Σ）：**

```
           SPY      IWM      EFA    ...    TLT      GLD
SPY     0.00012  0.00010  0.00009  ...  -0.00004  0.000001
IWM     0.00010  0.00015  0.00011  ...  -0.00003  0.000002
EFA     0.00009  0.00011  0.00014  ...  -0.00002  0.000003
...
TLT    -0.00004 -0.00003 -0.00002  ...   0.00008  0.000005
GLD     0.000001 0.000002 0.000003 ...   0.000005 0.00009
```

- **对角线**：每个资产自己和自己的协方差 = 自己的**方差**（波动率²）
- **非对角线**：两个资产之间的协方差（正 = 同向，负 = 反向）
- 矩阵关于对角线对称：`Cov(SPY, TLT) = Cov(TLT, SPY)`

### 2.3 组合波动率：矩阵乘法 `w^T · Σ · w`

```python
np.sqrt(np.dot(weights.T, np.dot(returns.cov() * 252, weights)))
```

**为什么不能直接加权平均波动率？**

因为资产之间有相关性，负相关的资产放一起会**抵消波动**，简单加权平均会高估真实风险。必须用协方差矩阵把相互影响算进去。

#### `weights.T` 是什么？——矩阵转置

转置就是把矩阵的行和列互换：

```
列向量（9行1列）：        行向量（1行9列）：
  ┌ 0.30 ┐              ┌                          ┐
  │ 0.25 │   转置 (.T)  │ 0.30  0.25  0.15  ...   │
  │ 0.15 │  ─────────→  └                          ┘
  │  ... │
  └ 0.05 ┘
```

> **NumPy 的小注意**：在 NumPy 里，1D 数组（shape 为 `(9,)`）执行 `.T` 实际上形状不变，
> 因为 NumPy 认为 1D 数组没有"行/列"之分。
> 但 `np.dot` 会根据运算规则自动处理维度，所以代码仍然正确。
> 数学上我们把它理解为"行向量"即可。

#### 从内到外拆解整个表达式

```python
np.sqrt(np.dot(weights.T, np.dot(returns.cov() * 252, weights)))
#  ④        ③                 ①                      ②
```

用 3 个资产简化演示，设 `w = [0.5, 0.3, 0.2]`（3×1 列向量）。

---

**① 最内层：`returns.cov() * 252` — 日协方差矩阵年化 → 年化协方差矩阵 Σ（3×3）**

```
日协方差矩阵 × 252 = 年化协方差矩阵 Σ

Σ = [[0.04,  0.02, -0.01],    ← 行/列分别代表 SPY、TLT、GLD
     [0.02,  0.09,  0.01],
     [-0.01, 0.01,  0.03]]
```

注意：`* 252` 是乘在协方差矩阵上的，不是最后乘的。所以后续所有计算结果都已经是年化数值。

---

**② `np.dot(Σ, weights)` — 3×3 矩阵 × 3×1 向量 → 3×1 向量**

矩阵乘法规则：结果的每一行 = Σ 的对应行与 w 做点积（逐元素相乘后求和）。

```
         Σ（3×3）              w（3×1）     结果（3×1）
┌  0.04   0.02  -0.01 ┐   ┌ 0.5 ┐   ┌ 0.04×0.5 + 0.02×0.3 + (-0.01)×0.2 ┐   ┌ 0.024 ┐
│  0.02   0.09   0.01 │ × │ 0.3 │ = │ 0.02×0.5 + 0.09×0.3 +  0.01×0.2  │ = │ 0.039 │
└ -0.01   0.01   0.03 ┘   └ 0.2 ┘   └-0.01×0.5 + 0.01×0.3 +  0.03×0.2  ┘   └ 0.004 ┘
```

**这个 3×1 结果向量的含义**：在当前权重下，每个资产对组合总风险的贡献程度（已把它与其他资产的联动关系考虑进去）。

- SPY 贡献 0.024：权重大且与其他资产正相关，风险贡献最大
- GLD 贡献 0.004：与其他资产负相关，抵消了部分风险

---

**③ `np.dot(weights.T, [0.024, 0.039, 0.004])` — 1×3 行向量 × 3×1 向量 → 标量**

`weights.T` 在数学上是 1×3 的行向量，与上一步的 3×1 结果相乘，得到一个标量：

```
w^T（1×3）        ×    Σw（3×1）         → 标量（组合年化方差）

[0.5,  0.3,  0.2] × [0.024, 0.039, 0.004]

= 0.5×0.024 + 0.3×0.039 + 0.2×0.004
= 0.0120 + 0.0117 + 0.0008
= 0.0245  ← 组合年化方差（已年化，因为 Σ 在①中已乘以 252）
```

---

**④ 最外层：`np.sqrt(0.0245)` → 0.1565 = 年化波动率 15.65%**

方差开平方根，得到标准差，也就是波动率：

```
年化方差 0.0245 → np.sqrt(0.0245) ≈ 0.1565 → 年化波动率 15.65%
```

---

**整个计算的维度变化总结：**

```
returns.cov() * 252     →  Σ，形状 (9, 9)
np.dot(Σ, weights)      →  形状 (9, 9) × (9,) = (9,)   每个资产的风险贡献向量
np.dot(weights.T, 上面)  →  形状 (9,)  × (9,) = 标量    组合年化方差
np.sqrt(标量)            →  标量                         组合年化波动率
```

## 三、随机生成 1000 个组合

```python
def calculate_efficient_frontier(returns, num_portfolios=1000):
    num_assets = len(returns.columns)
    results = np.zeros((3, num_portfolios))

    for i in range(num_portfolios):
        weights = np.random.random(num_assets)
        weights /= np.sum(weights)              # 归一化，使权重和为 1
        p_return, p_volatility = portfolio_performance(weights, returns)
        results[0, i] = p_return
        results[1, i] = p_volatility
        results[2, i] = p_return / p_volatility # 注意：没有减去无风险利率
    return results.T
```

### 关于 `results[2, i]` 夏普比率的计算说明

此处计算的是：

```python
results[2, i] = p_return / p_volatility  # 未减去无风险利率
```

而标准夏普比率公式应为：

```
夏普比率 = (年化收益率 - 无风险利率) / 年化波动率
```

**此处省略无风险利率的原因**：这 1000 个组合的计算目的是**绘制散点图，观察有效前沿的整体形状**。无风险利率是一个对所有组合都相同的固定常数（如 0.02），减不减它不影响各点之间的相对排列顺序，散点图的轮廓形状完全一致。

```
组合A 简化夏普：0.10 / 0.15 = 0.667
组合B 简化夏普：0.08 / 0.12 = 0.667

组合A 正确夏普：(0.10 - 0.02) / 0.15 = 0.533
组合B 正确夏普：(0.08 - 0.02) / 0.12 = 0.500

→ 两种算法下 A 均优于 B，排名顺序不变，颜色深浅的相对关系正确。
```

需要得到**精确最优权重**时，必须使用完整公式，见后续 `minimize` 中的 `negative_sharpe_ratio` 函数，那里正确地减去了 `risk_free_rate`。

## 四、数学优化：找到最优权重

### 4.1 目标函数：为什么取负数？

```python
def negative_sharpe_ratio(weights, returns, risk_free_rate):
    p_return, p_volatility = portfolio_performance(weights, returns)
    return -(p_return - risk_free_rate) / p_volatility
```

`scipy.optimize.minimize` **只能做最小化**（找最小值）。

```
我们的目标：最大化夏普比率
等价于：最小化（负的夏普比率）

夏普比率 = 0.8  →  负夏普 = -0.8
夏普比率 = 1.2  →  负夏普 = -1.2  ← 更小，minimize 会往这个方向走
```

### 4.2 `minimize` 做了什么？

```python
result = minimize(
    negative_sharpe_ratio,         # 目标函数：我想最小化这个
    num_assets * [1./num_assets],  # 初始权重：每个资产平均分（1/9）
    args=(returns, risk_free_rate),# 传给目标函数的固定参数
    method='SLSQP',                # 优化算法
    bounds=bounds,                 # 每个权重的范围 [0, 1]
    constraints=constraints        # 权重之和 = 1
)
```

`minimize` 的工作流程：

```
① 从初始权重出发（每个资产各 1/9）
② 计算当前权重下的负夏普比率（目标函数值）
③ 计算梯度：微调每个权重，看目标函数如何变化
④ 沿梯度下降方向调整权重
⑤ 检查约束是否满足（权重和=1，每个权重在[0,1]内）
⑥ 重复②~⑤，直到目标函数不再显著下降
⑦ 返回最终权重 result.x
```

### 4.3 SLSQP 梯度下降的原理

**SLSQP = Sequential Least Squares Programming（序列最小二乘规划）**

SLSQP 是一种基于梯度的优化算法，每一轮迭代将"带约束的非线性问题"近似为一个更容易求解的"带约束的二次规划问题"，逐步逼近最优解。理解它需要先明确三个概念。

---

#### 概念一：梯度（Gradient）

梯度是目标函数对每个输入变量的**偏导数组成的向量**，表示"当某个变量增大一点点时，函数值会如何变化"。

在我们的问题中，目标函数 `f(weights)` 输入是 9 个权重，梯度就是一个长度为 9 的向量：

```
梯度 ∇f = [∂f/∂w_SPY, ∂f/∂w_IWM, ∂f/∂w_EFA, ..., ∂f/∂w_GLD]

例：
∂f/∂w_SPY = -0.03  → SPY 权重增加 0.001，负夏普减少 0.00003（往好的方向走）
∂f/∂w_EEM = +0.02  → EEM 权重增加 0.001，负夏普增加 0.00002（往坏的方向走）
```

每个分量的符号和大小告诉你：这个资产的权重该增加还是减少，以及影响有多强。

---

#### 概念二：梯度方向

梯度向量指向的是函数值**上升最快**的方向。

因此，若目标是**最小化**函数，应沿**梯度的反方向**调整权重：

```
梯度方向     → 函数值上升最快（远离最优解）
负梯度方向   → 函数值下降最快（趋近最优解）✓
```

类比：你站在山上，梯度方向是最陡的上坡方向，沿负梯度走就是往最陡的下坡方向走。

---

#### 概念三：学习率（Learning Rate）

学习率控制每一步沿梯度方向走**多大的步子**，是一个正的小数：

```
新权重 = 旧权重 - 学习率 × 梯度

学习率太大 → 步子太大，可能跨过最优点，来回震荡不收敛
学习率太小 → 步子太小，需要非常多次迭代才能到达最优点
```

SLSQP 不需要手动设置学习率，它内部会自适应地决定每步的步长，这是它相比简单梯度下降更稳定的原因之一。

---

#### SLSQP 完整迭代流程

```
初始状态：w0 = [1/9, 1/9, ..., 1/9]（9 个资产均等分配）
          f(w0) = -0.61（负夏普比率）

第 1 轮：
  计算梯度 ∇f(w0) = [-0.03, -0.01, +0.02, ...]
  负梯度方向：SPY、TLT 权重应增加，EEM 权重应减少
  自适应步长调整权重，同时检查约束（权重和=1，每项∈[0,1]）
  得到 w1，f(w1) = -0.73

第 2 轮：
  在 w1 处重新计算梯度，继续调整
  得到 w2，f(w2) = -0.85

...

第 N 轮：
  f(wN) 与 f(wN-1) 差异极小（低于收敛阈值），停止迭代
  输出 wN = optimal_weights
```

收敛过程可视化：

```
负夏普比率（目标函数值）
     |
-0.6 |  ×（初始点 w0）
     |   ↘
-0.8 |     ×（w1）
     |       ↘
-1.0 |         × ×（w2, w3...逐渐收敛）
     |              ×（最优点 wN）
     └──────────────────────→ 迭代轮数
```

> SLSQP 每轮迭代都在约束条件下求解一个近似的二次规划子问题，相比普通梯度下降，
> 它能更精确地处理"权重和必须等于 1"这类等式约束，收敛速度更快、结果更准确。

## 六、关于 bounds 和权重为 0 的问题

```python
bound = (0.0, 1.0)
bounds = tuple(bound for asset in range(num_assets))
```

**`(0.0, 1.0)` 表示权重的下界是 0.0，上界是 1.0，0.0 本身是合法的。**

你的理解是对的：优化结果中某些资产的权重**可能恰好是 0，也可能无限趋近于 0**。

```
例：优化结果可能是
SPY:  0.3512
IWM:  0.0000   ← 精确为 0（小盘股历史夏普低，被完全排除）
EFA:  0.0000   ← 精确为 0
EEM:  0.0031   ← 趋近于 0（极小配比）
AGG:  0.2241
TLT:  0.1876
VNQ:  0.0000   ← 精确为 0
GLD:  0.1423
DBC:  0.0917
```

这在数学上完全合理，意思是"在历史数据下，这些资产加入组合反而降低了夏普比率，最优解是不配置它们"。

实际使用时需要注意：
- 这是基于**历史数据**的最优，未来不一定成立
- 权重为 0 的资产可能在其他时间段是重要的分散工具
- 真实配置时一般会加额外约束，比如每个资产权重至少 5%

## 六、整体流程总结

```
输入：9 个资产的历史日收益率 returns（DataFrame）
          ↓
① 计算协方差矩阵 returns.cov()（9×9）
          ↓
② 随机生成 1000 种权重组合 → 算出各自风险/收益 → 画散点图（近似有效前沿）
          ↓
③ SLSQP 优化：从均等权重出发，梯度下降最小化负夏普比率
   约束：权重和=1，每个权重 ∈ [0, 1]
          ↓
④ 输出最优权重 optimal_weights（某些资产可能为 0）
          ↓
⑤ 图上用红色★标出最优组合，直观对比其他组合
```

> **核心结论**：有效前沿不是找"最赚钱的资产"，而是找"最合理的资产配比"。
> 通过分散投资，可以在不降低收益的前提下显著降低风险——这就是分散化的价值。
